# Testing Guidelines for Backend (imagey-server)

When adding new features or modifying existing API endpoints, please adhere to the following testing strategy:

- **Contract Tests:** Ensure that the Pact contract tests generated by the frontend (Playwright tests) pass on the backend. This acts as the primary source of verification for the API.
- **Integration Tests:** If the contract tests do not sufficiently cover the logic (e.g., verifying database state, file storage deletion, or side effects), create a new integration test (similar to `LoginTest`) that exercises the endpoints. 
- **Unit Tests:** Do **NOT** write Unit Tests for features. Rely entirely on Contract Tests and Integration Tests.
